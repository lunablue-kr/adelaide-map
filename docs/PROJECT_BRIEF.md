> 외부 리뷰(중간 점검 · 디자인 점검)용 요약 문서. 2026-07-17 갱신 (중간 점검 리뷰 반영).
> 라이브: https://lunablue-kr.github.io/adelaide-map/ (영문 `?lang=en`)

## 1. 제품이 무엇인가

호주 애들레이드(Adelaide) **한 도시**에 특화된, **한국인 관점의 생활·정착 가이드 지도**.
Leaflet 기반 단일 웹페이지. 정적(런타임 API 호출 없음, 모든 데이터 내장). GitHub Pages 배포.

**핵심 차별점(moat) = 지역특화(애들레이드) × 한국인 관점 × 현지 맥락(Context).**
- 커버리지(핀 개수)로 구글맵과 경쟁하지 않음. 구글이 전 세계 규모라 못/안 하는 "장소별 현지 맥락"을 노림.
  - 예: 카페 = "한국인 많음·콘센트·평일 오전 한산", 러닝 = "새벽 추천·가로등·화장실", 부동산 = "한국마트 차 7분·밤에 밝음".
  - 애들레이드 하나면 수작업 맥락 유지가 가능 = 그게 정확히 차별점.
- 이미 부분 구현됨: 줌아웃 시 지역 성격 라벨(바이브 라벨), LGA별 현지 노트형 설명, 국가별 식료품점 태깅.

## 2. 기술 구조

- **스택**: 순수 HTML/CSS/JS + Leaflet(지도). 빌드 스텝 없음. 프레임워크 없음.
- **파일 분리**(토큰 효율 + 유지보수):
  - `index.html` — 뼈대(5KB)
  - `style.css` — 스타일(20KB)
  - `app.js` — 전 로직(~1,200줄)
  - `data-i18n.js` — 한/영 번역
  - `data-*.js` — 카테고리별 데이터(지오메트리·POI). 큰 데이터는 파일로 격리.
- **데이터**: 전부 정적. OSM(OpenStreetMap) + 공식 관공서 + 구글맵 교차검증으로 추출·가공해 리터럴 배열로 내장.
- **다국어**: 한국어 기본, `?lang=en` 영어. 딥링크 `?lga=<id>&lang=en&color=rent|crime`.
- **캐시버스팅**: 코드/데이터 배포 시 `?v=YYYYMMDD` 갱신.

## 4. 데이터 방법론

- **베이스 = OSM Overpass API** → **18개 LGA(지자체) 경계로 point-in-polygon 클립**(권역 밖 제거).
- **반드시 구글맵/공식 소스 교차검증**(사용자 지침). OSM 단독은 오탐·누락 둘 다 있음. 실제로:
  - 공원: `leisure=park`만으론 대형지 누락(Belair National Park는 nature_reserve, Bonython Park는 OSM 완전 부재) → 보강·수동 큐레이트.
  - 대학: OSM 부정확 → TEQSA/공식으로 손 큐레이트.
  - 경기장: Entertainment Centre·Memorial Drive는 OSM stadium 아님 → 웹검증 보강.
- **큐레이트 vs 벌크**: 소수 랜드마크(명소·시설·쇼핑·대학·병원)는 손 큐레이트. 다수 POI(식당·카페·마트 등)는 OSM 벌크 + 기준 필터.
- **한계**: 수백 개 POI 전수 검증은 1회 불가 → 공식 대조 + 피드백 크라우드소싱으로 점진 개선.

## 5. 디자인 시스템 (현재까지)

### 핀(마커)
- **원칙: 기호(글리프) = 카테고리(오버레이) / 색 = 하위 분류.** 예) 학교는 학사모 글리프 하나, 색으로 초/중/대 구분.
- **팔레트 7색 고정**: 빨·주·노·초·시안·파·보 (`#f87171 #fb923c #facc15 #4ade80 #22d3ee #60a5fa #c084fc`). 모든 핀 색은 이 안에서만. 기호가 카테고리를 구분하니 색 재사용은 무해.
- **줌 기반 표현**: 줌 ≤11 = 지역 성격 라벨(바이브 라벨) / <12 = 작은 점 / ≥12 = 기호 핀.

### 격리(하위 하나만 보기)
- **지도 마커**: 선택 외 = **완전 숨김**(opacity 0 + pointer-events none, 비인터랙티브) + 선택은 맨 앞으로. *(2026-07-17 Work Order A.3에서 0.22 투명도 → 완전 숨김으로 변경)*
- **범례/필터 칩**: 선택 외 = "테두리만" 표현.
  - 데스크톱 미니범례: 비선택 점 = 속 빈 색 링.
  - 모바일 하위칩: 비선택 = 텍스트를 감싼 얇은 테두리 박스(선택은 채운 박스).

### 모바일 크롬 (map-first)
- 상단바 `[피드백][검색][언어]`, 그 아래 **오버레이 칩바(가로스크롤)** + 필요 시 **하위 칩바**.
- 우측 세로 "지도 색상" 버튼 → 카테고리/렌트/치안 팝업 + 하단 스케일바.
- **바텀시트**: peek(이름+요약) ↔ 확장(상세) 스와이프 토글.
- 데스크톱은 좌측 통합 패널(검색/오버레이/색상/피드백) + 우측 미니범례.

### 기타
- LGA 채움은 옅게(0.11) + 경계선 강조 — 포인트 가독 우선.
- 별/하트 아이콘은 **향후 유저 즐겨찾기용으로 예약**(다른 데 쓰지 않음).

## 6. 히스토리 (요약)

1. **초기**: 단일 HTML 모놀리식(v6/v7) → 4파일 분리(index/style/app/data) → 이후 데이터 카테고리별 다분할.
2. **기반 데이터**: LGA 18 + 서버브 406 경계, 대중교통(기차/트램/버스), 학교(대학 큐레이트), 치안·렌트 choropleth.
3. **POI 오버레이 확장**: 마트 → 병원/의료 → 쇼핑 → 식당 → 카페 → 술집 → 명소/시설 분리 → **공원·레저** → **생활·행정(관공서/은행/우체국/통신/도서관)** → 명소에 **경기장/공연장** 추가.
4. **핀 시스템 전면 개편**: 기호=카테고리·색=하위 원칙, 7색 팔레트 확정, 줌 기반 점↔기호.
5. **모바일 크롬 전면 재구성**: map-first, 칩바, 바텀시트.
6. **최근 UX 수정**: 오버레이 칩바 가로스크롤 시 지도 팬 딸림 해결(touch-action), 격리 표현을 투명도→테두리 방식으로 정리.
7. **명소·시설 기본 OFF 전환** → 이제 상시표시는 서버브 경계뿐.

## 10. 맥락 태그 스키마 (로드맵 2단계 설계 원칙)

장소별 맥락 데이터 구조에 처음부터 포함할 필드:

```json
{
  "context": ["카공좋음", "콘센트넉넉"],
  "korean_tip": "주차장 진입로 좁음",
  "settlement_stage": 1,
  "verified": "2026-07"
}
```

- `verified`(검증 시점) 필드는 필수 — "낡음 방지"는 나중에 붙이기 가장 어려운 속성.
- **거리 감각의 한국인 앵커링**: "시티까지 15분" 대신 "한인마트까지 도보 X분" — 기존 마트 데이터로 자동 계산 가능, 큐레이션 비용 0.
- 노출 순서: 팝업 안 뱃지(칩) → 태그 축적 후 필터로 승격. 지도 표면 노출은 하지 않음.
- 크라우드소싱 입력을 받기 시작하는 시점부터 innerHTML 이스케이프 처리 필수.

## 11. 마케팅·배포 전략

원칙: **"지도 만들었어 + 링크"는 실패하는 포맷. 링크가 아니라 답과 콘텐츠를 던진다.**

1. 단톡방·카페에서 동네/렌트 질문에 지도 내용으로 답변 + 딥링크(`?lga=`) 첨부 — 광고가 아니라 답변이 되게.
2. 콘텐츠 변환: 스크린샷 + 인사이트 포스트("동네별 렌트 지도로 정리", "SA Police 치안 데이터 시각화") — 네이버 호주 카페, 애들레이드 한인 페이스북 그룹.
3. **영문판으로 r/Adelaide 포스트** — 단일 액션 중 기대수익 최고. 현지인 피드백 = 크라우드소싱 맥락 태그 첫 공급원.
4. 피드백은 부탁이 아니라 과제로: "집 구한다 치고 동네 3개 골라봐(3분)" / 지인 사용 장면 5분 관찰.
5. 시점 타겟: 수요층은 "이주·유학 준비 중" — 워홀/유학 카페, 대학 입학생 커뮤니티, 유학원.
6. 니치 도구는 느리게 붙는 게 정상. 반응은 감이 아니라 애널리틱스 숫자로 판단.

## 12. 디자인 결정사항 (리디자인 전 확정분)

- **오버레이 수·핀 수는 줄이지 않는다.** 성능 문제는 컬링/지연로딩으로, 인지 부하는 프리셋으로 해결.
- **클러스터링 기각** — 숫자 버블은 "동네의 성격"을 전달하지 못함. 대안: 뷰포트 컬링 + 밀도 높은 오버레이(식당·공원)만 기호 전환 임계 13~14로 차등.
- **라이트 테마 유력** — "필드 가이드" 톤엔 종이 느낌 뉴트럴 라이트가 문법에 맞음(+ 야외 가독성). 확정 전 타일 토글(CARTO Positron↔Dark Matter) 넣고 실사용 비교.
- **아이콘은 직접 그리지 않고 오픈소스 세트(Lucide/Phosphor/Tabler) path 사용** — 스트로크·시각 무게 일관성 확보. 리디자인 전 퀵윈 가능.
- **필드노트 톤 참고사항**: 형광펜/노트 질감 등 톤은 채택. 스티커/와펜형 이미지 핀은 기각(마커 3천 개 성능), 감성 사진 큐레이션 기각(라이선스·유지 부담, moat 아님).
- 브랜드 자산 = LGA 설명문·바이브 라벨의 편집자적 목소리. 리디자인 전에도 바이브 라벨/현지 노트에 세리프 계열 폰트, 팝업 맥락 문장에 "제작자 노트" 표기로 차별화 가능.
- 오버레이 13종 그룹핑(프리셋 전 임시): **살기**(서버브·교통·학교·의료·생활행정) / **먹고 놀기**(식당·카페·술집·명소) / **장보기·인프라**(마트·쇼핑·공원·시설).
- 격리 표현: 의미론(선택/흐림)은 플랫폼 공통, 표현은 플랫폼별 최적화 유지.
- 명소 하위 이질성(해변·와이너리 vs 경기장): 실해 없음, 여행자 프리셋 설계 때 자연 재분류.

## 13. 참고

- 데이터는 정적 스냅샷(OSM/SQM/SA Police/공식). 렌트·치안은 반기 재검증 필요 — 캘린더에 등록할 것.
- 의료·식당 등 벌크 OSM 데이터는 개별 미검증분 존재(크라우드소싱으로 개선 예정).
- 접근성, 오프라인, PWA 여부는 미논의.
- 애널리틱스: GoatCounter 적용 완료(페이지뷰 + 오버레이/필터/색상/LGA/언어/피드백 이벤트). 대시보드는 운영자 전용, `#skip-analytics`로 운영자 기기 제외.
